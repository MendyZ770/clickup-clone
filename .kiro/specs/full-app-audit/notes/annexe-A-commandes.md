# Annexe A — Journal des commandes exécutées

## Objet

Cette note constitue le **journal exhaustif des commandes shell** exécutées pendant l'audit `full-app-audit`. Elle est destinée à être incorporée telle quelle en **annexe A** de l'`audit-report.md` (cf. `design.md` > Components and Interfaces > Structure de `audit-report.md` > 5. Annexes > A) et satisfait l'exigence de traçabilité posée par `requirements.md` (clause 4.4).

Conventions :

- **Une ligne par commande** réellement exécutée pendant l'audit, dans l'ordre chronologique d'exécution.
- **Aucune commande destructive** : seules des commandes inspectives, de lecture ou de build non destructif (par exemple `git status --short`, `git diff --stat`, `npm run build`) sont consignées. Toute commande modifiant l'état du dépôt en dehors de `.kiro/specs/full-app-audit/` ou de `.next/` doit déclencher un Finding `F-LOG-NNN` Severity_High avant exécution.
- **Sortie tronquée** : si la sortie d'une commande est tronquée pour tenir dans la colonne `Résumé sortie`, la mention `[tronqué]` est ajoutée explicitement à la fin du résumé. Le log complet correspondant est, le cas échéant, conservé dans l'annexe dédiée (par exemple annexe B pour le `Build_Check`).
- **Durée** : exprimée en secondes (`s`) ou en minutes (`min`), arrondie à la précision utile.
- **Exit code** : code de sortie effectif observé dans le terminal (entier).
- **Référence croisée** : lorsque la sortie complète est consignée ailleurs (annexe B, note d'inventaire, etc.), le `Résumé sortie` renvoie au nom du fichier de destination.
